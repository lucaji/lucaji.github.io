StartRecApp does not need any account registration nor collects any of your personal data or User Content.

The App uses un-encrypted peer-to-peer  (P2P) data packets which relies on your Wifi-LAN  (Local Area Network) to communicate with each other StartRecApp device  (recorder). The App connects to Apple’s NTP time server  (time.apple.com) to achieve a better time synchronisation between connected recorders when using timed controls.


Recorder. A compatible iOS device running StartRecApp. When connected to a wifi network and published, they are usually referred as “Nearby Recorders”.


Publishing. In order to connect to other devices, StartRecApp needs to advertise its network service to other nearby recorders. The published service exchanges the following device specific information: Unique Identification Number, Model and Name. Once each StartRecApp instance has successfully published itself on the network, the discovery phase can take place.


Discovering. The network browser constantly queries the network infrastructure for changes within the StartRecApp service domain. This enable devices to be seen by others and start exchanging handshaking data such as the recorder’s identification data and the media gallery file list.


Media File. In this release of StartRecApp a valid media file is any standard JPG file with extensions either *.jpg or *.jpeg. Each file is available for Consolidation when the recorders connect each other. The recorders are said to be the source for their media file.


User Content. The media gallery containing the actual recorded media is stored only within StartRecApp’s Documents folder on each Recorder device. This folder is accessible via iTunes Sharing. Each take photograph is stored locally in JPEG format within its Take’s media file collection folder. The latter, in turn, belongs to its Session’s takes collection folder.


Local Media. The local media includes all the actual files you have stored in one Recorder’s storage either by taking a photo on the very device, or by copying it  (Consolidation) from another nearby recorder. Altogether, the local media can be accessed and previewed remotely by any connected nearby recorder.


Remote Media. This media is stored on another recorder in respect to the one you are referring at. If the remote source recorder becomes unreachable, all its media file will disappear as well. If you want to copy media then you can Consolidate the sessions.


Consolidation. You can initiate the session consolidation from the media gallery session information screen. This operation collects all the remote media file for each session folder from all the nearby source devices starting a copy of each missing file to be transferred onto the initiator storage space using your Wifi-LAN network.


Share. You are solely responsible to select the media to be shared ie. to email recipients, social apps you may have installed or by other means offered by the standard iOS sharing service, as the actual master file  (full resolution image) can be sent over.
