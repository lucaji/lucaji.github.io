Excel Object Model Techniques and patterns

Interfacing with Excel's object model could be quirky. These notes will depict some aspects of Office Add-In development.

Nested using Pattern

I am using this kind of pattern when accessing some actual Excel object: 

    		void StyleRedColorIfDifferent(Excel.Worksheet ws, string targetRange) {
                using (var aRange = ws.Range(targetRange))
                using (var theFont = aRange.Font) {
                    long theColor = Convert.ToInt64(theFont.Color);
                    if (theColor != 0xff)
            			theFont.Color = 0xff;
    			}
			}

With this pattern of accessing inner properties through nested usings I make sure all objects gets disposed correctly.
